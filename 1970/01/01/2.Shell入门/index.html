shell

命令行解释器（外壳程序）
centos的shell叫做bash

命令行的作用就是传递用户的输入输出

权限

  1. 用户：
  • root用户：超级管理员，不受权限约束
  • 普通用户：受权限约束的用户（一定要把root密码保护好）
  • 普通用户的密码最好不要和root密码一样，不然切换容易搞错
查看当前用户
whoami
切换root用户
su     #在本路径切换到root用户，路径不变 
su -   #相当于root用户重
新登录，路径变为root目录
退出当前用户
exit
Ctrl d
从root变为普通用户
su username  #这里不需要密码，所以非常危险
sudo命令
不切换用户，让用户直接执行root命令，输入的密码是用户密码
    sudo command #短暂提权，执行root命令
    sudo whoami  #这里发现用户是root
如果无法执行，需要将该用户添加到配置文件中，类似与白名单

什么是权限？

  • 约束某一个或某一类人（一个或一个群体）
  • 目标对象（电影网站，音乐等）本来就没有对应的属性（业务）
  • 一件事情能否允许，能被谁做
  • 文件u权限=人+文件属性
人（角色/身份）：
  1. 拥有者(owner):
  2. 所属组(grouper):
  3. 其他人(other):

文件属性：

  • r：读
  • w：写
  • x：执行
第一列的10个字符：
  1. 文件类型，win下可根据后缀来区分类型，linux下文件类型和后缀无关！（但是用户可以用后缀来区分），通过ll显示的众多熟悉中的第一列来区分文件类型，但是一些应用比如gcc需要特定的后缀，比如.c
  • - : 普通文件(源代码，库文件，可执行程序，压缩包等)
  • d ：目录文件
  • c : 字符设备文件：比如键盘，显示器（/dev/tty）
  • b ：块设备：磁盘（/dev/vdal）
  • l : 链接文件：比如win桌面的快捷方式
  • p :管道文件
  • s ：socket文件
  1. 剩下的9个3个一组，分别对应拥有者，所属组和other
  • 有（r/w/x）
  • 无（-）

如何操作权限（修改权限）

修改权限操作：chmod命令
    chmod u-r test.txt # u:user，去掉拥有者的读权限 
    chmod u+x test.txt # 给拥有者加上可执行权限
    chmod u-rwx test.txt #加上所有权限
    chmod g+r text #g：group，对所属组操作
    chmod o+r ... #其他用户
    chmod u+rwx，g+rwx，o-rwx test.txt #同时对所有人操作
    特殊： chmod a+rwx ... #a：all给所有人加权限
权限从左到右读
修改拥有者操作：chown,用户需要别人同意
    sudo chown name test.txt
修改用户组：chgrp
同时修改拥有者和所属组
    sudo chown name1:name2 test.text
同时，可以：“有”对应1,“无”对应”0”,可以转化为二进制（八进制方案）
    chmod 000 test.txt
    chmod 777 test.txt
    chmod 660 test.txt

file命令

    file 文件名 #查看文件类型
